## Description

Create an `InMemorySnapshotStore` class that implements the `SnapshotStore` Protocol using in-memory dict storage instead of the filesystem. This enables Relay users to create lightweight pipelines in tests without temporary directories. It stores snapshots in `dict[str, dict[str, ContextEnvelope]]` keyed by pipeline_id, and respects all the same error codes as `LocalFileSnapshotStore`.

### Task 01-02-A: Create InMemorySnapshotStore implementation

Create `src/relay/snapshot_in_memory.py`:

```python
"""In-memory snapshot store for Relay testing and lightweight pipelines.

Owns: in-memory snapshot storage, Protocol compliance verification.
Does NOT: persist data across process restarts, provide transactional guarantees, or support concurrent access.
"""
```

Implementation details:

- Module docstring (three-line format)
- `__all__ = ["InMemorySnapshotStore"]`
- Class `InMemorySnapshotStore` (does NOT extend `Closeable` or `Protocol` — structural subtyping)
- Internal storage: `_snapshots: dict[str, dict[str, ContextEnvelope]]` mapping `pipeline_id → {snapshot_id: envelope}`
- Internal index: `_index: dict[str, list[str]]` mapping `pipeline_id → [snapshot_id, ...]` (sorted by step)
- Method `save_snapshot(self, envelope: ContextEnvelope) -> Result[str]`:
  - Validates pipeline_id via `PIPELINE_ID_PATTERN` (import from `relay.envelope`) — return `Failure(INVALID_PIPELINE_ID)` on mismatch
  - Generates snapshot_id in format `{pipeline_id}@{step}_{uuid hex[:12]}` (same format as LocalFileSnapshotStore)
  - Stores envelope in `_snapshots[pipeline_id][snapshot_id]`
  - Maintains sorted index (by step extracted from snapshot_id)
  - Returns `Success(snapshot_id)`
- Method `load_snapshot(self, snapshot_id: str) -> Result[ContextEnvelope]`:
  - Validates snapshot_id format via `SNAPSHOT_ID_PATTERN` — return `Failure(INVALID_SNAPSHOT_ID)`
  - Looks up in `_snapshots[pipeline_id][snapshot_id]` — return `Failure(SNAPSHOT_NOT_FOUND)` if missing
  - Returns `Success(envelope)`
- Method `get_latest_snapshot(self, pipeline_id: str) -> Result[ContextEnvelope]`:
  - If pipeline_id not in index — return `Failure(PIPELINE_NOT_FOUND)`
  - If index entry empty — return `Failure(NO_SNAPSHOTS)`
  - Gets last snapshot_id from index, loads it, returns `Success(envelope)`
- Method `list_snapshots(self, pipeline_id: str) -> Result[list[str]]`:
  - Returns `Success([])` if pipeline_id not in index
  - Returns `Success(list of snapshot_ids)` in step-sorted order
- Method `close(self) -> None`:
  - Clears internal storage (`_snapshots.clear()`, `_index.clear()`)

Private helper methods:
- `_extract_step_from_snapshot_id(self, s_id: str) -> int` — extracts step number (same logic as module-level function in snapshot.py but needs to handle InvalidSnapshotIdError for sorting)
- `_validate_pipeline_id(self, pipeline_id: str) -> bool` — single use in save_snapshot

Import `PIPELINE_ID_PATTERN`, `SNAPSHOT_ID_PATTERN` from `relay.envelope` and `relay.snapshot` respectively.
Import `_extract_step_from_snapshot_id` from `relay.snapshot` for index sorting.
Import `uuid`, `datetime` for snapshot_id generation.
