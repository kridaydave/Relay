## 0. Executive Summary

v0.4 delivers **Layer 4 — Async Pipelines** from the Design Document. It adds a single new pipeline entry point — `execute_parallel_step()` — that lets framework builders run N agent adapters concurrently against the same context snapshot, then merge their outputs deterministically using one of three join strategies: **UNION**, **VOTE**, or **FIRST_WINS**.

The core architecture is unchanged. `ContextEnvelope` stays immutable and signed. The threading lock model is preserved — forks never write to shared state, only read from a captured snapshot. The existing `execute_step_with_manifest` remains the sole commit path.

**What ships:**
- `relay.parallel` package: `JoinStrategy`, `ForkSpec`, `ForkResult`
- `fork_runner.py` — stateless coroutine that runs one adapter against a snapshot
- `join.py` — UNION, VOTE, and FIRST_WINS join strategy implementations
- `ContextEnvelope` gains 4 optional fork-metadata fields (backward-compatible)
- `ValidationResult` gains `confidence_score: float` (used by VOTE)
- `CoreRelayPipeline.execute_parallel_step()` — new async entry point
- 4 new `ErrorCode` entries: `MERGE_CONFLICT`, `ALL_FORKS_FAILED`, `FORK_EXECUTION_FAILED`, `INVALID_JOIN_STRATEGY`
- Full unit + integration test coverage

**What does NOT ship (explicitly out of scope per Design Doc §4):**
- Streaming within a fork (post-v0.4)
- Nested parallelism (forks that themselves fork)
- Dynamic fork count at runtime (count fixed at call time)
- Per-fork token budget tracking (v0.5 audit log scope)
- Cross-pipeline fork coordination (each pipeline is independent)

## 1. Architectural Invariants v0.4 Must Preserve

Before the detailed plan, the constraints that rule every design decision:

| Invariant | How v0.4 preserves it |
|---|---|
| Every committed envelope is signed | Only `execute_step_with_manifest` commits; it always signs |
| Rollback is always available | Pre-fork state is never mutated during fork phase; failure = state unchanged |
| No shared mutable state during I/O | Forks receive immutable frozen dataclasses; no lock held during adapter calls |
| Sequential steps work unchanged | `execute_parallel_step` is purely additive; no existing method is modified |
| `mypy --strict` zero errors | All new types annotated; `Optional` fields typed with `X | None` |

## 2. Concurrency Model — Why No asyncio.Lock

**Current state:** `PipelineState._lock` is `threading.Lock`. `execute_step_with_manifest` is sync. `execute_step_with_runner` is async but only holds the lock briefly (not during I/O).

**For v0.4 parallel execution**, forks run via `asyncio.gather()`. The temptation is to migrate to `asyncio.Lock`. **Do not do this** — it would require converting every `with self._state.transaction()` to `async with`, touching all existing sync pipeline methods.

**Why the threading.Lock model works for asyncio.gather:**

`asyncio.gather` executes coroutines on a single event loop thread. They interleave at `await` points but never run simultaneously at the CPU level. The threading.Lock is held only during brief sync operations (envelope reads and commits), never during `await adapter.run(...)`. So:

1. No two coroutines can hold the lock at the same time (event loop is single-threaded)
2. No deadlock possible (forks don't acquire the lock; only the final commit does)
3. No starvation possible (forks don't contend for the lock at all)

**The execution pattern is:**

```
async execute_parallel_step():
    ┌─ threading.Lock acquired ─────────────────────┐
    │ read pre_fork_envelope                        │
    │ build fork slices (immutable, no state write) │
    │ run budget check                              │
    └─ threading.Lock released ─────────────────────┘
            │
    asyncio.gather(fork_A, fork_B, fork_C)  ← no lock held
            │
    ┌─ threading.Lock acquired ─────────────────────┐
    │ execute_step_with_manifest(merged_payload)    │
    │ (signs, validates, snapshots, advances state) │
    └─ threading.Lock released ─────────────────────┘
```

This is identical to how `execute_step_with_runner` works today. The only difference: multiple adapter calls happen before the commit instead of one.

### 5.2 `JoinStrategy` — `src/relay/parallel/types.py`

```python
"""Parallel execution types for Relay v0.4.

Owns: JoinStrategy enum, ForkSpec, ForkResult.
Does NOT: implement join logic, execute adapters, or manage pipeline state.
"""

from enum import Enum
from dataclasses import dataclass
from typing import Any, TYPE_CHECKING

from relay.types import Failure, Result

if TYPE_CHECKING:
    from relay.runners.protocol import AgentOutput
    from relay.validator import ValidationResult
    from relay.slicer.manifest import AgentManifest


class JoinStrategy(str, Enum):
    """Strategy for merging parallel fork outputs."""
    UNION      = "UNION"       # merge all passing forks; conflict → MergeConflictError + rollback
    VOTE       = "VOTE"        # accept fork with highest confidence_score; discard failures
    FIRST_WINS = "FIRST_WINS"  # accept first passing fork; cancel remaining calls


@dataclass(frozen=True)
class ForkSpec:
    """Specification for one fork in a parallel step.

    Owns: the adapter name and manifest that define one fork's execution.
    Does NOT: execute the adapter, hold results, or reference pipeline state.
    """
    adapter_name: str
    manifest: "AgentManifest"


@dataclass(frozen=True)
class ForkResult:
    """Result from one fork's execution and validation.

    Owns: the fork's identity, output, and validation result.
    Does NOT: write to pipeline state or reference the registry.

    A ForkResult is always created — even for failed forks — so join
    strategies have a complete picture of what happened.
    """
    fork_index: int                        # 0-based index in original fork_specs list
    adapter_name: str                      # for audit trail
    success: bool                          # True iff adapter + validation both passed
    agent_output: "AgentOutput | None"     # None if adapter raised
    validation: "ValidationResult | None"  # None if adapter raised; confidence_score lives here
    failure: "Failure | None"              # set when success=False
```

**Why `fork_index` instead of a UUID:** The fork index is deterministic and directly maps back to the original `fork_specs` list. A UUID would add noise without value — the spec itself is the identity. The parallel step's UUID lives on the resulting envelope, not on individual forks.

### 5.3 `ContextEnvelope` Extension — `src/relay/envelope.py`

Four new optional fields with `None` defaults. They MUST come after all existing required fields.

```python
@dataclass(frozen=True)
class ContextEnvelope:
    # --- existing fields (unchanged) ---
    relay_version: str
    pipeline_id: str
    step: int
    timestamp: datetime
    token_budget_used: int
    token_budget_total: int
    payload: dict[str, Any]
    manifest_hash: str
    signature: str

    # --- v0.4: parallel step metadata (None for sequential steps) ---
    fork_id: str | None = None            # UUID of the parallel step; None for sequential
    join_strategy: str | None = None      # JoinStrategy.value or None
    fork_count: int | None = None         # total forks launched
    forks_succeeded: int | None = None    # forks that passed validation
```

**Backward-compatible signature:** The signature is extended ONLY when any fork field is non-None. Sequential envelopes (all fork fields None) produce identical signatures to v0.3.3. Old snapshots load cleanly — their fork fields default to None, signature verifies with the non-fork format.

```python
def _compute_signature(envelope: ContextEnvelope, secret: str) -> str:
    payload_json = json.dumps(envelope.payload, sort_keys=True, separators=(",", ":"))
    base = (
        f"{envelope.relay_version}|{envelope.pipeline_id}|{envelope.step}|"
        f"{_canonical_timestamp(envelope.timestamp)}|{envelope.token_budget_used}|"
        f"{envelope.token_budget_total}|{envelope.manifest_hash}|{payload_json}"
    )
    if envelope.fork_id is not None:
        base += (
            f"|{envelope.fork_id}|{envelope.join_strategy}|"
            f"{envelope.fork_count}|{envelope.forks_succeeded}"
        )
    return hmac.new(secret.encode(), base.encode(), hashlib.sha256).hexdigest()
```

### 5.4 `ValidationResult` Extension — `src/relay/validator.py`

Add `confidence_score: float` to `ValidationResult`. This is the VOTE strategy's selection criterion.

```python
@dataclass(frozen=True)
class ValidationResult:
    """Result of validating a handoff between envelopes."""
    has_contradiction: bool
    diff: dict[str, Any]
    contradiction_details: str | None
    confidence_score: float = 1.0   # 0.0 = contradiction; 1.0 = fully faithful
```

**Confidence formula — computed inside `validate_handoff()`:**

```
if has_contradiction:
    confidence_score = 0.0
else:
    new_entities    = _extract_entities(current_envelope.payload)
    known_entities  = _extract_entities(previous_envelope.payload)
    total_new       = len(new_entities)
    if total_new == 0:
        confidence_score = 1.0   # no new entities → maximally conservative
    else:
        preserved       = len(new_entities & known_entities)
        confidence_score = preserved / total_new
```

**Interpretation:**
- `1.0` — every entity in the new output existed in the previous context (fully faithful)
- `0.5` — half the entities are new (uncertain)
- `0.0` — contradiction detected OR all entities are new

**Note:** This is an approximation (documented as such per Coding Rules §6.1). It uses the same `_extract_entities` already in the validator. No new heuristic is introduced.

**Backward compatibility:** `confidence_score` has a default of `1.0`. Existing code that constructs `ValidationResult(has_contradiction=..., diff=..., contradiction_details=...)` continues to work without passing `confidence_score`. Explicit construction in tests must be updated.

## 6. Fork Execution — `src/relay/parallel/fork_runner.py`

```python
"""Fork runner for Relay v0.4 parallel execution.

Owns: executing one adapter against a pre-fork envelope snapshot.
Does NOT: write to pipeline state, acquire locks, or know about join strategies.

Each call is a pure coroutine — the same pre_fork_envelope can be passed to
N concurrent _run_single_fork calls with no contention.
"""

import time
from typing import TYPE_CHECKING

from relay.parallel.types import ForkResult, ForkSpec
from relay.types import ErrorCode, Failure, Success
from relay.validator import HandoffValidator
from relay.envelope import ContextEnvelope, estimate_tokens
from relay.runners.protocol import ContextSlice, AgentOutput

if TYPE_CHECKING:
    from relay.runners.registry import AdapterRegistry
    from relay.slicer.manifest import AgentManifest


async def _run_single_fork(
    fork_index: int,
    spec: ForkSpec,
    slice_: ContextSlice,
    pre_fork_envelope: ContextEnvelope,
    registry: "AdapterRegistry",
    validator: HandoffValidator,
) -> ForkResult:
    """Execute one fork: run adapter, validate output, return ForkResult.

    Never raises. All errors are captured in ForkResult.success=False.
    Never writes to pipeline state — caller owns commit.

    Args:
        fork_index:        0-based index in the original fork_specs list.
        spec:              ForkSpec describing which adapter + manifest to use.
        slice_:            Pre-built ContextSlice for this fork (immutable).
        pre_fork_envelope: The snapshot envelope before the parallel step.
        registry:          Adapter registry to look up the adapter.
        validator:         Stateless HandoffValidator instance (shared across forks).

    Returns:
        ForkResult — always. success=True iff adapter ran and validation passed.
    """
    adapter_result = registry.get(spec.adapter_name)
    if isinstance(adapter_result, Failure):
        return ForkResult(
            fork_index=fork_index,
            adapter_name=spec.adapter_name,
            success=False,
            agent_output=None,
            validation=None,
            failure=adapter_result,
        )

    adapter = adapter_result.value

    try:
        agent_output = await adapter.run(slice_, spec.manifest)
    except Exception as e:
        return ForkResult(
            fork_index=fork_index,
            adapter_name=spec.adapter_name,
            success=False,
            agent_output=None,
            validation=None,
            failure=Failure(
                reason=f"Adapter '{spec.adapter_name}' raised: {type(e).__name__}: {e}",
                code=ErrorCode.FORK_EXECUTION_FAILED,
            ),
        )

    # Validate the fork output against the pre-fork snapshot.
    # Construct a temporary envelope for validation purposes only.
    # This envelope is NEVER committed — it exists solely for the diff.
    fork_payload = _agent_output_to_payload(agent_output)
    validation_result = validator.validate_handoff_payload(
        previous_envelope=pre_fork_envelope,
        new_payload=fork_payload,
    )
    if isinstance(validation_result, Failure):
        return ForkResult(
            fork_index=fork_index,
            adapter_name=spec.adapter_name,
            success=False,
            agent_output=agent_output,
            validation=None,
            failure=validation_result,
        )

    validation = validation_result.value
    passed = not validator.should_rollback(validation)

    return ForkResult(
        fork_index=fork_index,
        adapter_name=spec.adapter_name,
        success=passed,
        agent_output=agent_output,
        validation=validation,
        failure=None if passed else Failure(
            reason=validation.contradiction_details or "Fork validation failed",
            code=ErrorCode.FORK_EXECUTION_FAILED,
        ),
    )


def _agent_output_to_payload(output: AgentOutput) -> dict:
    """Shape AgentOutput into a dict for validation purposes (not for commit)."""
    raw: dict = {"text": output.text, **output.structured}
    if output.tool_calls:
        raw["tool_calls"] = output.tool_calls
    return raw
```

**Design note — `validate_handoff_payload`:** The existing `HandoffValidator.validate_handoff` takes two `ContextEnvelope` objects. For fork validation, we don't have a completed envelope for the fork output — it's just a payload dict. We need a new method `validate_handoff_payload(previous_envelope, new_payload)` that accepts a raw payload dict instead of a full envelope. This avoids creating a fake signed envelope just for validation.

Add to `HandoffValidator`:
```python
def validate_handoff_payload(
    self,
    previous_envelope: ContextEnvelope,
    new_payload: dict,
) -> Result[ValidationResult]:
    """Validate a raw payload dict against the previous envelope.

    Used by fork runner to validate fork output without creating a full envelope.
    Identical logic to validate_handoff() but accepts raw payload instead of envelope.

    REQUIRES: no lock — reads only immutable inputs.
    """
    # Delegate to existing internals; only the payload source changes.
    return self._validate_payloads(
        previous_payload=previous_envelope.payload,
        new_payload=new_payload,
    )
```

Internally this means extracting the core validation logic from `validate_handoff` into `_validate_payloads(previous_payload, new_payload)` and having `validate_handoff` call it. This is a pure refactor of the validator — no behavior change.

## 7. Join Strategies — `src/relay/parallel/join.py`

```python
"""Join strategy implementations for Relay v0.4 parallel execution.

Owns: merging ForkResults into a single payload per join strategy.
Does NOT: execute adapters, commit to pipeline state, or manage locks.
"""
```

### 7.1 UNION

All forks must succeed. Their payloads are merged. Any key written by two forks with different values → `MERGE_CONFLICT` + the whole parallel step fails (caller's state is unchanged).

```python
def _apply_union(fork_results: list[ForkResult]) -> Result[dict]:
    """Merge all passing forks. Conflict on any shared key with differing values.

    Per Design Doc §4: 'Merge all outputs; conflicts raise MergeConflictError + rollback.'
    Under UNION, a single failed fork fails the entire parallel step — all or nothing.

    Returns: Success(merged_payload) or Failure(MERGE_CONFLICT | ALL_FORKS_FAILED)
    """
    failed = [r for r in fork_results if not r.success]
    if failed:
        reasons = "; ".join(
            f"fork[{r.fork_index}] ({r.adapter_name}): {r.failure.reason}"
            for r in failed if r.failure
        )
        return Failure(
            reason=f"UNION: {len(failed)} fork(s) failed — {reasons}",
            code=ErrorCode.ALL_FORKS_FAILED,
        )

    merged: dict = {}
    conflicts: list[str] = []

    for result in fork_results:
        assert result.agent_output is not None  # success=True guarantees this
        fork_payload = _agent_output_to_payload(result.agent_output)
        for key, value in fork_payload.items():
            if key in merged and merged[key] != value:
                conflicts.append(key)
            elif key not in merged:
                merged[key] = value

    if conflicts:
        return Failure(
            reason=f"UNION merge conflict on keys: {sorted(set(conflicts))}",
            code=ErrorCode.MERGE_CONFLICT,
        )

    return Success(merged)
```

### 7.2 VOTE

Accept the fork with the highest `confidence_score` among passing forks. Failed forks are silently discarded. If all forks fail, return `ALL_FORKS_FAILED`.

```python
def _apply_vote(fork_results: list[ForkResult]) -> Result[dict]:
    """Accept the passing fork with the highest confidence_score.

    Per Design Doc §4: 'Accept highest-confidence passing fork.'
    Failed forks are discarded; only passing forks compete.

    Returns: Success(winner_payload) or Failure(ALL_FORKS_FAILED)
    """
    passing = [r for r in fork_results if r.success and r.validation is not None]
    if not passing:
        return Failure(
            reason=f"VOTE: all {len(fork_results)} forks failed",
            code=ErrorCode.ALL_FORKS_FAILED,
        )

    winner = max(passing, key=lambda r: r.validation.confidence_score)  # type: ignore[union-attr]
    assert winner.agent_output is not None
    return Success(_agent_output_to_payload(winner.agent_output))
```

### 7.3 FIRST_WINS

Accept the first passing fork. Cancel remaining in-flight tasks immediately. Failed forks are discarded.

```python
async def _apply_first_wins(
    fork_index_coros: list[tuple[int, ForkSpec, "Coroutine[Any, Any, ForkResult]"]],
) -> Result[dict]:
    """Accept the first passing fork; cancel the rest.

    Per Design Doc §4: 'Accept first passing fork; cancel remaining calls.'
    Cancellation is best-effort — tasks already in flight may complete naturally.

    Returns: Success(winner_payload) or Failure(ALL_FORKS_FAILED)
    """
    tasks: list[asyncio.Task[ForkResult]] = [
        asyncio.create_task(coro, name=f"fork-{idx}")
        for idx, _, coro in fork_index_coros
    ]
    winner_payload: dict | None = None
    pending = set(tasks)

    while pending and winner_payload is None:
        done, pending = await asyncio.wait(pending, return_when=asyncio.FIRST_COMPLETED)
        for task in done:
            result: ForkResult = task.result()
            if result.success and result.agent_output is not None:
                winner_payload = _agent_output_to_payload(result.agent_output)
                break

    # Cancel remaining in-flight tasks.
    for task in tasks:
        if not task.done():
            task.cancel()
    # Await cancelled tasks to suppress CancelledError propagation.
    await asyncio.gather(*tasks, return_exceptions=True)

    if winner_payload is None:
        return Failure(
            reason=f"FIRST_WINS: all {len(tasks)} forks failed or were cancelled",
            code=ErrorCode.ALL_FORKS_FAILED,
        )
    return Success(winner_payload)
```

**Why coroutines are passed in, not tasks:** For UNION and VOTE, `asyncio.gather` creates tasks internally. For FIRST_WINS, we must create tasks explicitly to cancel them. Passing coroutines into `_apply_first_wins` and creating tasks there keeps the strategy implementations symmetric and avoids double task creation.

### 7.4 Public Entry Point

```python
async def apply_join_strategy(
    strategy: JoinStrategy,
    fork_specs: list[ForkSpec],
    fork_results: list[ForkResult],  # used by UNION and VOTE
    first_wins_coros: list | None,   # used by FIRST_WINS only
) -> Result[dict]:
    """Route to the correct strategy implementation.

    Returns: Success(merged_payload) or Failure with MERGE_CONFLICT / ALL_FORKS_FAILED.
    """
    if strategy == JoinStrategy.UNION:
        return _apply_union(fork_results)
    elif strategy == JoinStrategy.VOTE:
        return _apply_vote(fork_results)
    elif strategy == JoinStrategy.FIRST_WINS:
        assert first_wins_coros is not None
        return await _apply_first_wins(first_wins_coros)
    else:
        return Failure(
            reason=f"Unknown join strategy: {strategy!r}",
            code=ErrorCode.INVALID_JOIN_STRATEGY,
        )
```

**Note on FIRST_WINS vs UNION/VOTE fork execution:** UNION and VOTE run all forks to completion (via `asyncio.gather`) before joining. FIRST_WINS cancels after the first pass. This asymmetry is intentional and mirrors the design doc. `execute_parallel_step` handles this:
- For UNION/VOTE: `asyncio.gather(*fork_coroutines)` → collect `ForkResult` list → pass to strategy
- For FIRST_WINS: pass raw coroutines to `_apply_first_wins` (it creates tasks internally)

## 8. Failure Handling

Matches Design Doc §4 table exactly:

| Strategy | Any fork fails | All forks fail | Merge conflict |
|---|---|---|---|
| **UNION** | Entire step fails → `ALL_FORKS_FAILED` → state unchanged | Same | `MERGE_CONFLICT` → state unchanged |
| **VOTE** | Failed fork discarded; compete on passing forks | `ALL_FORKS_FAILED` → state unchanged | N/A (no merge) |
| **FIRST_WINS** | Failed fork discarded; next completing fork wins | `ALL_FORKS_FAILED` → state unchanged | N/A (no merge) |

**"State unchanged"** means: `execute_parallel_step` returns `Failure` without calling `execute_step_with_manifest`. The pipeline's `current_envelope` is still the pre-fork envelope. No snapshot is written. No rollback needed — the mutation never happened.

### 9.1 New Method: `execute_parallel_step`

```python
async def execute_parallel_step(
    self,
    fork_specs: list[ForkSpec],
    join_strategy: JoinStrategy,
) -> Result[ContextEnvelope]:
    """Execute N adapter forks in parallel, merge via join strategy, commit one result.

    Pipeline sequence:
      1. Validate inputs (at least one spec, registry configured).
      2. Acquire lock: capture pre-fork envelope, build slices, check budgets.
      3. Release lock.
      4. For UNION/VOTE: asyncio.gather all forks → ForkResult list.
         For FIRST_WINS: pass coroutines to _apply_first_wins for task management.
      5. Apply join strategy → merged payload or Failure.
      6. On Failure: return immediately (state unchanged).
      7. On Success: call execute_step_with_manifest(merged_payload) to commit.
      8. Attach fork metadata to committed envelope.

    Args:
        fork_specs:     One ForkSpec per fork. Must be non-empty. Order is preserved
                        (fork_index in ForkResult matches list position).
        join_strategy:  How to merge results. See JoinStrategy enum.

    Returns:
        Success(envelope) — with fork metadata fields populated.
        Failure — if input validation, all forks, or merge fails. State unchanged.
        RollbackSuccess — if the validator triggers rollback during commit (unusual
                          but possible; treated identically to the sequential path).

    Raises:
        Nothing. All errors returned as Failure.
    """
    if not fork_specs:
        return Failure(
            reason="fork_specs must be non-empty",
            code=ErrorCode.INVALID_JOIN_STRATEGY,
        )
    if self.registry is None:
        return Failure(
            reason="No AdapterRegistry configured on this pipeline",
            code=ErrorCode.NO_REGISTRY,
        )

    # 1. Capture pre-fork state under lock (brief — no I/O).
    with self._state.transaction() as pre_fork_envelope:
        fork_slices = [
            self._build_context_slice(pre_fork_envelope, spec.manifest)
            for spec in fork_specs
        ]
        for spec, slice_ in zip(fork_specs, fork_slices):
            budget_result = self._check_budget(spec.manifest, pre_fork_envelope)
            if isinstance(budget_result, Failure):
                return budget_result

    assert pre_fork_envelope is not None or True  # may be None on step 0; fork of empty is allowed

    # 2. Execute forks (lock NOT held — I/O happens here).
    parallel_id = uuid.uuid4().hex
    fork_coros = [
        _run_single_fork(
            fork_index=i,
            spec=spec,
            slice_=slice_,
            pre_fork_envelope=pre_fork_envelope,
            registry=self.registry,
            validator=self._handoff_validator,
        )
        for i, (spec, slice_) in enumerate(zip(fork_specs, fork_slices))
    ]

    if join_strategy == JoinStrategy.FIRST_WINS:
        merged_result = await _apply_first_wins(
            [(i, spec, coro) for i, (spec, coro) in enumerate(zip(fork_specs, fork_coros))]
        )
        fork_results: list[ForkResult] = []  # not collected for FIRST_WINS
        forks_succeeded = 1 if isinstance(merged_result, Success) else 0
    else:
        fork_results = list(await asyncio.gather(*fork_coros))
        forks_succeeded = sum(1 for r in fork_results if r.success)
        merged_result = apply_join_strategy(join_strategy, fork_specs, fork_results, None)

    if isinstance(merged_result, Failure):
        return merged_result

    # 3. Commit the merged payload through the existing pipeline (re-acquires lock internally).
    merged_payload = merged_result.value
    commit_result = self.execute_step_with_manifest(merged_payload, manifest=None)

    # 4. Attach fork metadata to the committed envelope.
    if isinstance(commit_result, Success):
        envelope_with_meta = commit_result.value.with_fork_metadata(
            fork_id=parallel_id,
            join_strategy=join_strategy.value,
            fork_count=len(fork_specs),
            forks_succeeded=forks_succeeded,
        )
        # Re-sign after adding fork metadata.
        from relay.envelope import compute_signature
        signed = envelope_with_meta.with_signature(
            compute_signature(envelope_with_meta, self._context_broker.signing_secret)
        )
        # Update state with the re-signed envelope (brief lock acquisition).
        with self._state.transaction():
            self._state.set_current(signed)
        return Success(signed)

    return commit_result  # Failure or RollbackSuccess pass through unchanged
```

**Design note — re-signing after fork metadata:** `execute_step_with_manifest` commits an envelope without fork metadata (it doesn't know about parallel steps). We attach metadata after commit and re-sign. The state update uses `set_current` (not `archive_and_set`) because the envelope at this step is being refined, not superseded. This is a one-line update — no new snapshot is written; the metadata is considered part of the committed step's record.

**Alternative considered and rejected — pass fork metadata into `execute_step_with_manifest`:** This would require adding optional parameters to `execute_step_with_manifest`, modifying the sync method signature, and threading fork metadata through `_handle_initial_step` / `_handle_subsequent_step`. The post-commit refinement approach above keeps the commit path unchanged and is locally correct.

## 11. Validator Refactor Required

Before implementing the fork runner, `HandoffValidator.validate_handoff` must be refactored to expose `_validate_payloads`. Current internal structure:

```python
# Current (schematic)
def validate_handoff(previous_envelope, current_envelope):
    # ... computes diff, detects hallucination, computes confidence ...
```

**Required refactor (Commit 1 gate):**

```python
def _validate_payloads(
    self,
    previous_payload: dict,
    new_payload: dict,
) -> Result[ValidationResult]:
    """Core validation logic operating on raw payloads.

    REQUIRES: no lock — reads only immutable inputs. Stateless.
    """
    diff = self._compute_diff(previous_payload, new_payload)
    hallucination = self._detect_hallucination(previous_payload, new_payload)
    has_contradiction = self._has_critical_key_loss(previous_payload, new_payload) or hallucination.exceeds_threshold
    contradiction_details = hallucination.details if has_contradiction else None

    new_entities = self._extract_entities(new_payload)
    known_entities = self._extract_entities(previous_payload)
    if not new_entities:
        confidence_score = 1.0
    elif has_contradiction:
        confidence_score = 0.0
    else:
        preserved = len(new_entities & known_entities)
        confidence_score = preserved / len(new_entities)

    return Success(ValidationResult(
        has_contradiction=has_contradiction,
        diff=diff,
        contradiction_details=contradiction_details,
        confidence_score=confidence_score,
    ))

def validate_handoff(
    self, previous_envelope: ContextEnvelope, current_envelope: ContextEnvelope
) -> Result[ValidationResult]:
    return self._validate_payloads(previous_envelope.payload, current_envelope.payload)

def validate_handoff_payload(
    self, previous_envelope: ContextEnvelope, new_payload: dict
) -> Result[ValidationResult]:
    return self._validate_payloads(previous_envelope.payload, new_payload)
```

This is a pure refactor — no behavior change. Existing tests remain valid. `confidence_score` defaults to `1.0` for any `ValidationResult` constructed without it (backward compat).

## 14. Explicit Out-of-Scope for v0.4

| Item | Reason | Target |
|---|---|---|
| Per-fork token budget tracking in audit log | Audit log is v0.5 scope | v0.5 |
| Nested parallelism (fork of a fork) | Doubles complexity; no documented use case | post-v1.0 |
| Dynamic fork count at runtime | Fork specs must be provided upfront; runtime expansion is speculative | post-v1.0 |
| Streaming within a fork | Streaming adapters not designed yet | post-v0.4 |
| Cross-pipeline fork coordination | Each pipeline is independent; cross-pipeline coordination is orchestration, not Relay | never |
| PARTIAL_UNION (accept some forks on conflict) | Not in design doc; conflict should be explicit failure | never |
| Sync `execute_parallel_step` | Concurrent I/O requires async; a sync wrapper would serialize forks, defeating the purpose | never |
| Fork retry logic | Relay never owns retries (Design Doc Appendix B §2) | never |
| Weighted VOTE (per-fork confidence multiplier) | Adds config surface area; the validator's score is already the weight | post-v1.0 |
